## General (Allgemein)

| Tastenkombination | Funktion |
|-------------------|----------|
| `Ctrl+Shift+P`, `F1` | **Command Palette anzeigen** - Zugriff auf alle VS Code-Befehle |
| `Ctrl+P` | **Quick Open** - Schnell zu Datei navigieren |
| `Ctrl+Shift+N` | **Neues Fenster/Instanz** öffnen |
| `Ctrl+Shift+W` | **Fenster/Instanz schließen** |
| `Ctrl+,` | **Benutzereinstellungen** öffnen |
| `Ctrl+K Ctrl+S` | **Tastenkürzel-Einstellungen** öffnen |

> **Hinweis für Assistenten:** `Ctrl+Shift+P` ist der wichtigste Befehl - öffnet die Command Palette für Zugriff auf alle Funktionen.

| Tastenkombination | Funktion |
|-------------------|----------|

| Tastenkombination | Funktion |
|-------------------|----------|

| Tastenkombination | Funktion |
|-------------------|----------|
| `Shift+Alt+(Maus ziehen)` | **Spalten-Auswahl** (Box-Selection) |
| `Ctrl+Shift+Alt+(Pfeiltasten)` | **Spalten-Auswahl mit Tastatur** |
| `Ctrl+Shift+Alt+PgUp/PgDn` | **Spalten-Auswahl Seite hoch/runter** |

> **Productivity-Boost:** Multi-Cursor ist eines der mächtigsten Features - ermöglicht gleichzeitige Bearbeitung an mehreren Stellen.

## Tipps für effiziente Tastaturnutzung

### Chord-Shortcuts verstehen

Viele Shortcuts sind "Chords" (Akkorde) - zwei Tastenfolgen nacheinander:
- **Beispiel:** `Ctrl+K Ctrl+S` bedeutet: Erst `Ctrl+K` drücken, dann `Ctrl+S`
- Beide Tastenkombinationen müssen innerhalb kurzer Zeit gedrückt werden

### Eigene Shortcuts anpassen

- **`Ctrl+K Ctrl+S`** öffnet Keyboard Shortcuts-Editor
- Alle Shortcuts können angepasst werden
- Eigene Keybindings in `keybindings.json` definieren

### Shortcuts lernen

1. **Command Palette nutzen:** `Ctrl+Shift+P` zeigt Shortcuts neben Befehlen
2. **Häufig genutzte zuerst:** Konzentrieren Sie sich auf Ihre Workflow-Shortcuts
3. **Schrittweise erweitern:** 1-2 neue Shortcuts pro Woche lernen

### Plattformunterschiede

- **Windows:** Diese Anleitung
- **Mac:** `Cmd` statt `Ctrl`, `Option` statt `Alt`
- **Linux:** Meist wie Windows, einige Abweichungen möglich

## Zusammenfassung für KI-Assistenten

### Kategorisierung der Shortcuts:

- **Essentiell (täglich):** Ctrl+P, Ctrl+S, Ctrl+F, Ctrl+Shift+P
- **Navigation:** F12, Ctrl+T, Ctrl+G, Alt+←/→
- **Bearbeitung:** Alt+↑/↓, Ctrl+D, Ctrl+/, Shift+Alt+F
- **Multi-Cursor:** Alt+Click, Ctrl+Shift+L, Shift+Alt+I
- **Debug:** F5, F9, F10, F11
- **Layout:** Ctrl+\, Ctrl+B, F11

### Workflow-Empfehlungen:

1. **Quick Navigation:** Ctrl+P → Dateiname eingeben → Enter
2. **Multi-Edit:** Ctrl+F → Alt+Enter → alle Vorkommen bearbeiten
3. **Code-Explore:** Ctrl+T → Symbol suchen → F12 zur Definition
4. **Refactoring:** F2 für Umbenennung, Ctrl+. für Quick Fix

### Wichtigste Unterschiede zu anderen Editoren:

- **Command Palette** (`Ctrl+Shift+P`) zentraler als in vielen anderen Editoren
- **Quick Open** (`Ctrl+P`) kombiniert mehrere Funktionen (Datei, Symbol, etc.)
- **Chord-Shortcuts** (`Ctrl+K` gefolgt von zweitem Shortcut) sind häufig
- **Multi-Cursor** ist sehr ausgereift und zentral
